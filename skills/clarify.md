# Clarify Skill

Use this skill when an issue, discussion, reopen, or mention is real but still ambiguous.

Goal:

- turn loose conversation into a bounded problem statement
- expose missing information quickly
- keep questions short and answerable in GitHub comments

Apply this skill when:

- the request mixes multiple problems
- the desired outcome is unclear
- a reopened item has no fresh explanation
- contributors are debating direction without a concrete next step

Expected output:

- a 3-5 line brief of the actual problem
- at most 3 pointed questions
- 2-3 plausible paths if tradeoffs exist
- one recommended smallest next step

Guardrails:

- do not invent product decisions
- do not ask broad exploratory questions when one concrete blocker is enough
- if the ambiguity is architectural or product-defining, escalate instead of deciding
